Variables Locales y Globales

Concepto de Variables Locales y Globales

Ahora que sabes cómo crear funciones que encierran bloques de código, tenemos que hablar de un concepto importante: variables locales frente a variables globales.

Cuando declaramos una nueva variable dentro de una función, decimos que es local para esa función. Eso es porque solo esa función puede ver esa variable. El resto del programa fuera de la función no puede verla. Una vez que estamos fuera de esa función, es como si ya no existiera.

Cada lenguaje de programación es diferente, pero para JavaScript, es importante saber que las variables tienen un "ámbito de la función": una función puede ver las variables locales que fueron declaradas dentro de ella y las variables globales que fueron declaradas fuera de ella, pero no puede ver las variables locales dentro de otras funciones.

Ejemplo

En la siguiente función de ejemplo, resultado es una variable local:

function sumar(num1, num2) {
var resultado = num1 + num2;
document.write("El resultado local es: " + resultado);
return resultado;
}
sumar(5, 7);
document.write(resultado); // ¡ERROR, la variable es local!

Cuando ejecutemos ese código, obtendremos un error en la última línea: "resultado is not defined". La variable solo está definida dentro de la función, porque es donde la declaramos con la línea var resultado = y no está definida fuera de ella.

Cuando declaramos una variable fuera de nuestras funciones, decimos que es una variable global. Esto es porque todas las funciones pueden acceder a ella y hacer lo que quieran con ella.

var globalResult;

function sumar(num1, num2) {
       globalResult = num1 + num2;
       document.write("El resultado global es: " + globalResult);
}
sumar(5, 7);
document.write(globalResult);

Cuando ejecutemos el código del ejemplo de arriba, no obtendremos ningún error, porque declaramos globalResult fuera de la función, así que podemos acceder a ella cuando queramos.
